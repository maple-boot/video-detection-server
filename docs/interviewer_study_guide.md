# 面试官快速学习路线 — 图形算法校招面试

> **目标：** 用最短时间掌握面试题涉及的核心知识点，做到能听懂候选人回答、判断是否达标。
> **定位：** 不需要你成为专家，只需要理解每个知识点的"关键概念"和"判断标准"。
> **预计学习时间：** 3-5 小时（集中学习）

---

## 学习策略

```
每个知识点的学习模式：
1. 它是什么？（一句话定义）
2. 为什么重要？（解决什么问题）
3. 面试中怎么判断候选人水平？（达标线）
```

## 模块一：深度学习基础（30 分钟）

### 1.1 卷积操作

**一句话理解：** 卷积就是用一个小的"滑动窗口"（卷积核）在图像上滑动，每个位置做乘加运算，提取局部特征。

**你需要知道的核心概念：**

| 概念 | 解释 | 面试判断 |
|------|------|----------|
| 卷积核 | 一个小矩阵（如 3x3），通过滑动提取特征 | 候选人应能解释"局部感受野" |
| Padding | 在输入边缘补零，控制输出尺寸 | 知道"same"和"valid"的区别 |
| Stride | 卷积核每次滑动的步长 | stride=2 会使输出尺寸减半 |
| 通道 | RGB 图像有 3 个通道，卷积核也有对应通道数 | 理解多通道卷积的计算方式 |
| 1x1 卷积 | 不改变空间尺寸，只做通道间信息融合 | 加分项：提到"降维""增加非线性" |
| 深度可分离卷积 | MobileNet 的核心，分两步减少计算量 | 加分项：能算出计算量减少比例 |

**快速验证公式：** 输出尺寸 = (输入 - 卷积核 + 2×padding) / stride + 1

**判断标准：**
- ✅ 达标：能解释卷积是什么 + 会算输出尺寸
- 🌟 优秀：能解释 1x1 卷积作用、深度可分离卷积原理

---

### 1.2 过拟合与正则化

**一句话理解：** 过拟合 = 模型把训练数据"背"下来了，遇到新数据就不行了。正则化 = 各种防止"死记硬背"的方法。

**你需要知道的核心概念：**

| 概念 | 解释 | 面试判断 |
|------|------|----------|
| 过拟合 | 训练好、测试差 | 基本概念，必须知道 |
| Dropout | 训练时随机"关掉"一些神经元 | 知道训练/推理行为不同 |
| L1/L2 正则化 | 在损失函数中加入权重惩罚项 | L1 产生稀疏解，L2 让权重更小 |
| 数据增强 | 对训练数据做变换（翻转、旋转、裁剪）扩充数据量 | 知道常用增强方法 |
| 早停 | 验证集效果不再提升时停止训练 | 简单实用的方法 |

**判断标准：**
- ✅ 边标：能说出 2-3 种防止过拟合的方法
- 🌟 优秀：能解释每种方法的原理和适用场景

---

### 1.3 Batch Normalization (BN)

**一句话理解：** BN 就是让每一层的输入都"标准化"到均值为0、方差为1的分布，这样训练更稳定更快。

**你需要知道的核心概念：**

| 概念 | 解释 |
|------|------|
| BN 的作用 | 加速收敛、缓解梯度消失、正则化效果 |
| 训练时 | 用当前 batch 的均值和方差 |
| 推理时 | 用训练过程中积累的滑动平均均值和方差 |
| BN 放在哪 | 通常放在卷积层之后、激活函数之前 |

**判断标准：**
- ✅ 达标：知道 BN 的作用 + 训练/推理的区别
- 🌟 优秀：能解释为什么 BN 能加速收敛

---

### 1.4 梯度消失与梯度爆炸

**一句话理解：** 反向传播时，梯度经过很多层后要么变得极小（消失）要么变得极大（爆炸），导致网络无法正常训练。

**你需要知道的：**
- 梯度消失 → 深层网络前面的层学不动 → 解决：残差连接、BN
- 梯度爆炸 → 权重更新太大导致训练不稳定 → 解决：梯度裁剪、BN

**判断标准：**
- ✅ 达标：知道这两个问题是什么 + 各一个解决方法
- 🌟 优秀：能解释残差连接为什么能缓解梯度消失

---

### 1.5 损失函数与优化器

**一句话理解：** 损失函数告诉模型"你错得有多离谱"，优化器告诉模型"怎么改才能错得少"。

**你需要知道的核心概念：**

| 概念 | 解释 |
|------|------|
| 交叉熵损失 | 分类任务的标准损失函数 |
| Focal Loss | 解决样本不均衡，降低"容易样本"的权重 |
| SGD | 随机梯度下降，经典但收敛慢 |
| Adam | 自适应学习率，收敛快但有时泛化差 |
| 学习率 | 控制每次更新的步长，太大震荡、太小收敛慢 |

**判断标准：**
- ✅ 达标：知道交叉熵 + Focal Loss 的作用
- 🌟 优秀：能解释 Focal Loss 的公式原理（`(1-p)^γ` 调节因子）

## 模块二：经典视觉模型（30 分钟）

### 2.1 CNN 经典网络演进

**一句话理解：** 从 AlexNet 到 ResNet，网络越来越深，精度越来越高，关键创新解决了"深度增加反而效果变差"的问题。

**演进脉络（记住这条线就够了）：**

```
AlexNet (2012) → VGGNet (2014) → GoogLeNet (2014) → ResNet (2015)
   ↓                ↓                  ↓                  ↓
 开创性的         更深更规整         Inception模块        残差连接
 第一个CNN        用3x3卷积         多尺度特征           解决退化问题
```

**ResNet 核心（最重要）：**
```
普通网络：输入 x → 卷积层 → 输出 F(x)
ResNet：  输入 x → 卷积层 → 输出 F(x) + x  ← 多了一个"捷径连接"
```
- 为什么有效：梯度可以通过捷径直接回传，不会消失
- 效果：可以训练 100+ 层的网络而不退化

**判断标准：**
- ✅ 达标：能说出 ResNet 的残差连接是什么 + 为什么有用
- 🌟 优秀：能说出整个演进脉络 + 每个网络的创新点

---

### 2.2 目标检测

**一句话理解：** 目标检测 = 分类 + 定位，不仅要识别图中有什么，还要框出在哪里。

**两阶段 vs 一阶段（必须理解）：**

```
两阶段（Faster R-CNN）：
  图像 → 特征提取 → RPN生成候选区域 → 对每个候选区域分类 → 输出
  特点：精度高，速度慢（~5 FPS）

一阶段（YOLO）：
  图像 → 特征提取 → 直接预测框和类别 → 输出
  特点：速度快（30+ FPS），精度稍低
```

**核心概念：**

| 概念 | 解释 | 一句话理解 |
|------|------|-----------|
| Anchor（锚框） | 预先设定的不同大小和比例的框 | "模板框"，模型在模板基础上微调 |
| IoU | 两个框的交集面积 / 并集面积 | 衡量两个框"重叠程度" |
| NMS | 去掉重叠的冗余框，只保留最好的 | "去重"操作 |
| mAP | 所有类别的平均精度 | 检测模型的"总成绩" |
| FPN | 特征金字塔，多尺度检测 | 让模型能同时检测大目标和小目标 |

**Anchor-free（了解即可）：**
- 不用预设锚框，直接预测目标中心点或边界
- 代表：FCOS、CenterNet
- 优点：更简洁，不需要调 anchor 参数

**判断标准：**
- ✅ 达标：知道两阶段/一阶段区别 + NMS 作用
- 🌟 优秀：能解释 RPN、Anchor、FPN 的原理

---

### 2.3 语义分割

**一句话理解：** 给图像中每个像素分类，输出一张和原图同样大小的"分类地图"。

**核心概念：**

| 概念 | 解释 |
|------|------|
| 语义分割 vs 目标检测 | 分割是像素级，检测是目标级（框） |
| FCN | 全卷积网络，用卷积替代全连接层，输出分割图 |
| 编码器-解码器 | 编码器缩小特征图提取语义，解码器放大恢复细节 |
| U-Net | 医学图像常用，有"跳跃连接"保留细节 |
| 上采样 | 把小特征图放大回原图尺寸（双线性插值、转置卷积） |

**判断标准：**
- ✅ 达标：知道分割和检测的区别 + FCN 基本思想
- 🌟 优秀：能解释 U-Net 结构、编码器-解码器思想

## 模块三：Transformer 与视觉 Transformer（20 分钟）

### 3.1 Transformer 基础

**一句话理解：** Transformer 的核心是"自注意力机制"——让模型能关注到输入中任意位置之间的关系。

**你需要知道的：**

```
传统 CNN：只能看到局部区域（感受野有限）
Transformer：能同时看到所有位置（全局注意力）
```

**核心概念：**

| 概念 | 解释 |
|------|------|
| Self-Attention | 计算序列中每个元素与其他所有元素的相关性 |
| Multi-Head Attention | 多个注意力头并行，捕捉不同类型的关系 |
| 位置编码 | 给每个位置加一个"坐标信息"，否则 Transformer 不知道顺序 |

**判断标准：**
- ✅ 达标：知道 Transformer 能"全局关注"，比 CNN 感受野大
- 🌟 优秀：能解释 Self-Attention 的 QKV 机制

---

### 3.2 Vision Transformer (ViT)

**一句话理解：** 把图像切成小块（patch），每个小块当作一个"词"，然后用 Transformer 处理。

**流程（记住这个就够了）：**

```
图像 (224x224)
  ↓ 切成 16x16 的小块 → 得到 14x14 = 196 个 patch
  ↓ 每个 patch 展平 + 线性映射 → 得到 196 个 token
  ↓ 加上位置编码
  ↓ 送入 Transformer Encoder
  ↓ 分类输出
```

**ViT vs CNN：**
- ViT 需要大量数据训练（ImageNet 级别以上），小数据集效果不好
- ViT 捕获全局信息更好，CNN 局部特征提取更高效
- 现在趋势：两者结合（如 Swin Transformer）

**判断标准：**
- ✅ 达标：知道 ViT 把图像切成 patch 处理
- 🌟 优秀：能解释 Patch Embedding + 位置编码的作用

## 模块四：模型部署基础（15 分钟）

### 4.1 模型格式与部署工具

**一句话理解：** 训练用 PyTorch，部署时要转成更高效的格式，用专门的推理引擎跑。

**你需要知道的：**

| 工具/格式 | 用途 | 一句话理解 |
|-----------|------|-----------|
| ONNX | 通用模型交换格式 | "通用翻译器"，PyTorch→ONNX→各种引擎 |
| TensorRT | NVIDIA GPU 推理加速 | 把模型优化到极致，速度快 2-5 倍 |
| OpenVINO | Intel CPU 推理加速 | Intel 平台专用 |
| NCNN | 移动端推理 | 手机上跑模型用的 |

**判断标准：**
- ✅ 达标：知道 ONNX + TensorRT 是什么
- 🌟 优秀：能说出 TensorRT 的加速原理（量化、层融合、kernel调优）

---

### 4.2 模型量化

**一句话理解：** 把模型的数字精度降低（如从32位降到8位），用更少的内存和计算量，速度更快但可能损失一点精度。

**精度对比：**

```
FP32（32位浮点）→ 标准精度，训练用
FP16（16位浮点）→ 速度×2，精度几乎无损
INT8（8位整数） → 速度×4，需要校准，有精度损失
```

**判断标准：**
- ✅ 达标：知道 FP32/FP16/INT8 的区别 + 量化能加速
- 🌟 优秀：知道量化需要校准，了解 PTQ 和 QAT

## 模块五：场景分析题的判断技巧（15 分钟）

### 5.1 如何判断候选人的分析思路

**场景题没有标准答案，关键是看分析思路。判断标准：**

| 层次 | 表现 | 判断 |
|------|------|------|
| **优秀** | 先分析原因，再给方案，方案有优先级 | 有系统性思维 |
| **良好** | 能给出合理方案，但没有分析过程 | 有知识但缺逻辑 |
| **及格** | 只能给出一个方向（如"换大模型"） | 知识面窄 |
| **不及格** | 说不出任何思路 | 基础不扎实 |

### 5.2 小目标检测场景的参考答案

**原因分析方向：**
1. 输入分辨率太低 → 小目标缩放后丢失信息
2. 特征图尺度问题 → 小目标在深层特征图上占比太小
3. 标注质量问题 → 小目标标注可能不准确

**解决方案方向：**
1. 增大输入分辨率（640→1280）
2. 使用 FPN 多尺度特征
3. 图像切片检测（SAHI）
4. 数据增强（小目标复制粘贴）
5. 使用针对小目标优化的模型

**判断：** 候选人能说出 2-3 个方向就达标，能分析原因+给方案就优秀。

---

### 5.3 白天/夜间效果差异场景的参考答案

**原因分析方向：**
1. 训练数据分布不均（白天多、夜间少）
2. 夜间图像质量差（噪声、低亮度、模糊）
3. 模型没有学到光照不变特征

**解决方案方向：**
1. 增加夜间训练数据
2. 数据增强（亮度、对比度、噪声）
3. 图像预处理（亮度增强、去噪）
4. 使用更大的模型

**判断：** 候选人能想到"数据分布"问题就达标，能给出具体增强方案就优秀。

## 面试时的追问技巧

### 好的追问方式
- "你刚才提到了 X，能展开说说为什么吗？"
- "如果不用这个方法，还有什么替代方案？"
- "这个方法有什么缺点或局限性？"
- "在 XX 场景下，你觉得这个方法还适用吗？"

### 不好的追问方式
- ❌ "XX 的公式是什么？"（太死记硬背）
- ❌ "XX 有多少层？"（纯记忆，没有意义）
- ❌ 连续追问同一个知识点的细节（会让候选人紧张）

### 追问深度控制
- 候选人答得好 → 追问 1-2 层，看深度
- 候选人答得一般 → 换个知识点，看广度
- 候选人答不上来 → 给提示或换题，不要僵持
